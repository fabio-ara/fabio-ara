# Manual do Repositório de Perfil

## 1. Finalidade

Este repositório existe para exibir o `README.md` principal no topo do perfil público do GitHub.

Para isso funcionar corretamente no GitHub:

- o repositório deve ter exatamente o mesmo nome do usuário, `fabio-ara`;
- o repositório deve ser público;
- o arquivo `README.md` deve ficar na raiz e conter conteúdo publicado.

## 2. Direção editorial

O perfil deve apresentar o autor de forma acadêmica e profissional, com sobriedade e sem exagero promocional.

O texto deve deixar claros:

- a graduação em ADS no IFSP;
- a atuação profissional na CETESB com desenvolvimento de soluções internas e automação de processos de negócio;
- o interesse principal em desenvolvimento full-stack;
- os projetos autorais ligados a tecnologia educacional, autoria pedagógica, compilação de conteúdo e linguagem.

## 3. Elementos recomendados no GitHub

### 3.1 Bio

Sugestão de bio curta para o campo do perfil:

`ADS no IFSP | CETESB: desenvolvimento de soluções internas e automação de processos de negócio | Full-stack, NLP e tecnologia educacional`

### 3.2 Repositórios fixados

Ordem sugerida de itens fixados no perfil:

1. `AraLearn`
2. `AraLearn Factory`
3. `SPOLOG1`
4. `Disassembly`
5. `SPOPFDS`

### 3.3 Informações públicas

Recomenda-se manter públicos, quando fizer sentido:

- nome;
- foto;
- localização;
- vínculo com o IFSP;
- link do LinkedIn.

Evitar publicar dados pessoais desnecessários.

## 4. Regra de manutenção

- o `README.md` deve evitar referências muito datadas, para não envelhecer rápido;
- mudanças relevantes de foco acadêmico, profissional ou de projetos devem ser refletidas no `README.md`;
- a ordem dos repositórios fixados pode ser revista conforme o portfólio evoluir.
